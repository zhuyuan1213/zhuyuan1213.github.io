[2021 鹤城杯]easyecho

1、前言

利用canary报错，调用stack_chk_fail函数，他会打印argv[0],此时我们利用stack smash来打印出bss上的flag

2、题目

检查一下保护机制

发现保护全开，到ida中看下程序流程

注意v9是函数sub_CF0的指针。查看一下这个函数

这个函数的作用是把flag读取到unk_202040处的bss段上。

所以程序执行执行了这个函数之后，把flag读到bss段，我们只需要利用stack smash将flag打印出来即可。

但是执行这个函数的前提是第二次输入的也是backdoor，而且第一个打印函数正好能让我们泄露出sub_CF0函数的地址。

缓冲区是衔接的，所以第一个打印函数会连带着v9一起打印出来。

最后需要将argv[0]存放的地址修改为bss段上flag的真正地址。

所以需要调试计算argv[0]和输入时缓冲区的偏移，计算出来是0x168。

    p.sendlineafter(a, b)

    print(p.recv())

    p.recvuntil(a)
def debug():
    gdb.attach(p)
    pause()
def get_addr():
    return u64(p.recvuntil(b'\x7f')[-6:].ljust(8, b'\x00'))

context(os='linux', arch='amd64', log_level='debug')

# leak pro_base
